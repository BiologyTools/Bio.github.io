All the analyzer Nu\+Get packages produced in this repo support \+\_\+.\+editorconfig based analyzer configuration\+\_\+. End users can configure the behavior of specific CA rule(s) OR all configurable CA rules by specifying supported key-\/value pair options in an {\ttfamily .editorconfig} file. You can read more about {\ttfamily .editorconfig} format \href{https://editorconfig.org/}{\texttt{ here}}.\hypertarget{md_packages__microsoft__code_analysis__analyzers_3_3_3_documentation__analyzer__configuration_autotoc_md873}{}\doxysubsection{.\+editorconfig format}\label{md_packages__microsoft__code_analysis__analyzers_3_3_3_documentation__analyzer__configuration_autotoc_md873}
Analyzer configuration options from an {\ttfamily .editorconfig} file are parsed into {\itshape general} and {\itshape specific} configuration options. General configuration enables configuring the behavior of all CA rules for which the provided option is valid. Specific configuration enables configuring each CA rule ID or CA rules belonging to each rule category, such as {\ttfamily Naming}, {\ttfamily Design}, {\ttfamily Performance}, etc. or CA rules with a specific custom tag, such as {\ttfamily Dataflow}. Our options are {\itshape case-\/insensitive}. Below are the supported formats\+:


\begin{DoxyEnumerate}
\item General configuration option\+:
\begin{DoxyEnumerate}
\item {\ttfamily dotnet\+\_\+code\+\_\+quality.\+Option\+Name = Option\+Value}
\end{DoxyEnumerate}
\item Specific configuration option\+:
\begin{DoxyEnumerate}
\item {\ttfamily dotnet\+\_\+code\+\_\+quality.\+Rule\+Id.\+Option\+Name = Option\+Value}
\item {\ttfamily dotnet\+\_\+code\+\_\+quality.\+Rule\+Category.\+Option\+Name = Option\+Value}
\item {\ttfamily dotnet\+\_\+code\+\_\+quality.\+Rule\+Custom\+Tag.\+Option\+Name = Option\+Value}
\end{DoxyEnumerate}
\end{DoxyEnumerate}

\item General configuration option\+:
\begin{DoxyEnumerate}
