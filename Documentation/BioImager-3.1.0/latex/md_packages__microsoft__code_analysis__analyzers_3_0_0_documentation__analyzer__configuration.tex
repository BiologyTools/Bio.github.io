Starting with version {\ttfamily 2.\+6.\+3}, all the analyzer Nu\+Get packages produced in this repo, including the Fx\+Cop Analyzers Nu\+Get package, support \+\_\+.\+editorconfig based analyzer configuration\+\_\+. End users can configure the behavior of specific CA rule(s) OR all configurable CA rules by specifying supported key-\/value pair options in an {\ttfamily .editorconfig} file. You can read more about {\ttfamily .editorconfig} format \href{https://editorconfig.org/}{\texttt{ here}}.\hypertarget{md_packages__microsoft__code_analysis__analyzers_3_0_0_documentation__analyzer__configuration_autotoc_md842}{}\doxysection{.\+editorconfig format}\label{md_packages__microsoft__code_analysis__analyzers_3_0_0_documentation__analyzer__configuration_autotoc_md842}
Analyzer configuration options from an .editorconfig file are parsed into {\itshape general} and {\itshape specific} configuration options. General configuration enables configuring the behavior of all CA rules for which the provided option is valid. Specific configuration enables configuring each CA rule ID or CA rules belonging to each rule category, such as \textquotesingle{}Naming\textquotesingle{}, \textquotesingle{}Design\textquotesingle{}, \textquotesingle{}Performance\textquotesingle{}, etc. or CA rules with a specific custom tag, such as \textquotesingle{}Dataflow\textquotesingle{}. Our options are {\itshape case-\/insensitive}. Below are the supported formats\+:
\begin{DoxyEnumerate}
\item General configuration option\+:
\begin{DoxyEnumerate}
\item {\ttfamily dotnet\+\_\+code\+\_\+quality.\+Option\+Name = Option\+Value}
\end{DoxyEnumerate}
\item Specific configuration option\+:
\begin{DoxyEnumerate}
\item {\ttfamily dotnet\+\_\+code\+\_\+quality.\+Rule\+Id.\+Option\+Name = Option\+Value}
\item {\ttfamily dotnet\+\_\+code\+\_\+quality.\+Rule\+Category.\+Option\+Name = Option\+Value}
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item {\ttfamily dotnet\+\_\+code\+\_\+quality.\+Rule\+Custom\+Tag.\+Option\+Name = Option\+Value}
\end{DoxyEnumerate}
\end{DoxyEnumerate}

For example, end users can configure the analyzed API surface for analyzers using the below {\ttfamily api\+\_\+surface} option specification\+:
\begin{DoxyEnumerate}
\item General configuration option\+:
\begin{DoxyEnumerate}
\item {\ttfamily dotnet\+\_\+code\+\_\+quality.\+api\+\_\+surface = public}
\end{DoxyEnumerate}
\item Specific configuration option\+:
\begin{DoxyEnumerate}
\item {\ttfamily dotnet\+\_\+code\+\_\+quality.\+CA1040.\+api\+\_\+surface = public}

\item {\ttfamily dotnet\+\_\+code\+\_\+quality.\+Dataflow.\+api\+\_\+surface = public}
