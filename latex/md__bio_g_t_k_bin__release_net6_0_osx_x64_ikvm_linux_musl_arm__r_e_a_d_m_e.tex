IKVM is an implementation of Java for the Microsoft .NET platform. It can be used to quickly and easily\+:


\begin{DoxyItemize}
\item Execute compiled Java code (bytecode) on .NET Framework or .NET Core
\item Convert bytecode to a .NET assembly to directly access its API in a .NET project
\end{DoxyItemize}

These tasks can be done {\bfseries{without porting source code}} to .NET.\hypertarget{md__bio_g_t_k_bin__release_net6_0_osx_x64_ikvm_linux_musl_arm__r_e_a_d_m_e_autotoc_md1970}{}\doxysubsection{IKVM Components}\label{md__bio_g_t_k_bin__release_net6_0_osx_x64_ikvm_linux_musl_arm__r_e_a_d_m_e_autotoc_md1970}

\begin{DoxyItemize}
\item A Java virtual machine (JVM) implemented in .NET
\item A .NET implementation of the Java class libraries
\item A tool that translates Java bytecode (JAR files) to .NET IL (DLL or EXE files).
\item Tools that enable Java and .NET interoperability
\item A full JRE/\+JDK 8 runtime image.
\end{DoxyItemize}\hypertarget{md__bio_g_t_k_bin__release_net6_0_osx_x64_ikvm_linux_musl_arm__r_e_a_d_m_e_autotoc_md1971}{}\doxysubsection{Run Java Applications with .\+NET}\label{md__bio_g_t_k_bin__release_net6_0_osx_x64_ikvm_linux_musl_arm__r_e_a_d_m_e_autotoc_md1971}

\begin{DoxyEnumerate}
\item {\bfseries{Statically\+:}} By compiling a Java application into a .NET assembly using {\ttfamily \texorpdfstring{$<$}{<}Maven\+Reference\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}Ikvm\+Reference\texorpdfstring{$>$}{>}} or {\ttfamily ikvmc}.
\begin{DoxyItemize}
\item Libary assemblies can be referenced by any .NET application with a compatible target framework and platform. Types can be referenced by using the Java package name like a .NET namespace.
\item Executable assemblies can be launched by specifying the class containing the {\ttfamily main()} method to execute at runtime when building using {\ttfamily ikvmc}.
\end{DoxyItemize}
\item {\bfseries{Dynamically\+:}} By running a Java application using the {\ttfamily java} executable inside of the JDK Runtime Image. The Java bytecode is converted on-\/the-\/fly to CIL and executed. The experience should be identical to a normal JDK.
\end{DoxyEnumerate}\hypertarget{md__bio_g_t_k_bin__release_net6_0_osx_x64_ikvm_linux_musl_arm__r_e_a_d_m_e_autotoc_md1972}{}\doxysection{What IKVM is Not}\label{md__bio_g_t_k_bin__release_net6_0_osx_x64_ikvm_linux_musl_arm__r_e_a_d_m_e_autotoc_md1972}

\begin{DoxyItemize}
\item A converter utility to transform Java source code to C\# source code
\item A decompiler utitity to transform compiled Java bytecode to C\# source code
\item A tool that runs .NET code in Java -\/ all IKVM conversions are Java \texorpdfstring{$>$}{>} .NET
\end{DoxyItemize}\hypertarget{md__bio_g_t_k_bin__release_net6_0_osx_x64_ikvm_linux_musl_arm__r_e_a_d_m_e_autotoc_md1973}{}\doxysection{Support}\label{md__bio_g_t_k_bin__release_net6_0_osx_x64_ikvm_linux_musl_arm__r_e_a_d_m_e_autotoc_md1973}

The IKVM project recommends that people do not redistribute FOSS Java libraries compiled with IKVM over public systems such as Nu\+Get.\+org, unless you are the original owner of that software and have a compelling reason.

Creating copies of FOSS Java libraries and publishing them to distribution mechanisms such as Nu\+Get.\+org creates eco-\/system confusion and dependency conflicts downstream. We provide a system so that .NET users of Java libraries can reference those libraries directly out of the standard Java ecosystem mechanisms\+: Maven Central, etc though IKVM.\+Maven. Remember, very few libraries exist in a vacuum. Libraries often depend on dozens of other libraries. Two unrelated Java libraries often depend on the same underlying Java library. A complex method of dependency conflict resolution and version unification has to be involved in resolving this hierarchy for any individual downstream project. You are likely going to be introducing duplicate classes into the users of your versions, or causing your users to depend upon the wrong version of other libraries.

In addition to the above, we presently do not guarentee that API between statically compiled assemblies and the IKVM.\+Java and IKVM.\+Runtime libraries remains stable. A statically compiled assembly built against one version of IKVM is not guarenteed to run against a higher version of IKVM, even at the patch level. We would like to advertise better support for this in the future, but at present there is a lot of work planned for the IKVM.\+Runtime and IKVM.\+Java APIs that we don\textquotesingle{}t want to inhibit. Since we also provide build tools to make the process of recompiling assemblies from the build machine easy, if recommendations are followed, this should be of limited impact.

There are exceptions to this advice, such as the library not being published to Maven. In that case, guidance would be to advocate that the original Java library in fact be published to Maven, or do the work yourself, as that is the appropriate place for Java libraries.

We cannot force you to conform to this advice. But for the health of the ecosystem, we urge you to take it under consideration yourself.
